## 一、当前状态概述

`InkRenderer.lua` 的 `IMAGE_PATHS` 表注册了 **13 张** 128×128 透明 PNG 贴图（8 资源 + 4 线索 + 1 撤离点），存放于 `assets/image/` 下按类别分子目录，全部被渲染代码实际使用。

**渲染架构**：贴图优先 + 矢量 fallback 双保险
- `InkRenderer.initImages(nvg)` 在 `Start()` 中加载所有 IMAGE_PATHS 贴图
- `drawResource()` / `drawClue()` / `drawEvacPoint()` 先检查 `imgHandles[key]`，有则用贴图，无则回退矢量绘制
- 矢量 fallback 可作为贴图加载失败时的降级方案

**贴图加载入口**：`scripts/render/InkRenderer.lua` → `InkRenderer.initImages(nvg)`
**贴图存放目录**：`assets/image/`（按 `items/` `clues/` `beasts/` `players/` `environment/` `evacuation/` 分类）

## 十、贴图制作规范

新增贴图时遵循以下规范：

1. **格式**：PNG，透明背景（alpha 通道）
2. **尺寸**：统一 128×128px（地图实体图标）
3. **风格**：水墨白描 / 国风手绘 / 宣纸质感，颜色参考 `InkPalette.lua`
4. **命名规则**：`<类型>_<名称>_<日期时间戳>.png`
5. **存放路径**：`assets/image/<类别>/`（items / clues / beasts / players / environment / evacuation）
6. **代码注册**：在 `InkRenderer.lua` 的 `IMAGE_PATHS` 表中添加 key → path 映射
7. **加载方式**：`InkRenderer.initImages(nvg)` 在 `Start()` 中自动加载所有注册贴图
