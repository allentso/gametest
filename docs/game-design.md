## 一、游戏概述

**类型**：水墨风异兽捕获 Roguelike（单局制提取玩法）
**视角**：2D 俯视角
**画面**：全程 NanoVG 矢量渲染，水墨白描 + 朱砂点缀 + 宣纸底色
**平台**：竖屏移动端
**单局时长**：8 分钟

**核心循环**：
```
大厅准备 → 进入灵境探索 → 收集资源 → 调查线索 → 触发异兽 → 压制QTE → 封灵捕获 → 撤离带走 → 结算入账
```

**一句话描述**：你是一名封灵师，在限时 8 分钟内深入迷雾笼罩的灵境，追踪异兽线索、捕获珍稀灵兽，并在天地崩塌前撤离——带不走的，就永远留在灵境了。

## 四、地图与探索

### 4.1 地图生成

- **地图尺寸**：20×30 格（竖屏方向，宽20格 × 高30格）
- **出生点**：底部中央 `(10, 2)`，周围 3×3 区域强制为小路
- **撤离点**：4 个，分布在地图边缘（四角附近）

### 4.2 地形类型

| 地形 | 可通行 | 特殊效果 | 视觉 |
|------|--------|---------|------|
| 草地 `grass` | ✅ | 无 | 稀疏兰草弧线，翡翠底色 |
| 小路 `path` | ✅ | 无 | 暖色带状路面 + 踏痕 |
| 岩石 `rock` | 部分阻挡（35-45%） | 大石不可通行 | 卵石点 + 皴纹 / 巨石轮廓 |
| 竹林 `bamboo` | 部分阻挡（50-60%） | 密竹不可通行 | 截面圆 + 放射叶 |
| 水面 `water` | ❌ | 完全不可通行 | 蓝色波纹 + 涟漪 + 光斑 |
| 瘴气 `danger` | ✅ | 无（视觉警告） | 暗红漩涡 + 朱砂散点 |
| 边界墙 `wall` | ❌ | 地图边框 | 浓墨填充 + 皴纹 |

### 4.3 区域分布（按深度）

| 区域 | Y 范围 | 特征 |
|------|--------|------|
| 安全区 | 距边缘 ≤3 格 | 35% 小路、50% 草地、15% 竹林，资源简单 |
| 搜索区 | 深度 ≤40% | 混合地形，开始出现岩石阻挡 |
| 稀有区 | 深度 40%-70% | 更多岩石/竹林，10% 瘴气地形，兽魂/镇灵砂开始出现 |
| 高危区 | 深度 >70% | 25% 瘴气，密集障碍物，天晶/归魂符仅此处出现 |

### 4.4 资源分布

地图生成时放置 **25 个资源节点**，每个产出 1-3 单位。分布权重：

| 资源 | 权重 | 出现条件 |
|------|------|---------|
| 灵石 | 60 | 全图 |
| 追迹灰 | 20 | 全图 |
| 兽魂 | 10 | 深度 >40% |
| 天晶 | 5 | 深度 >70% |
| 镇灵砂 | 8 | 深度 >40% |
| 归魂符 | 5 | 深度 >70% |

### 4.5 迷雾系统（战争迷雾）

三种状态：
- **黑暗**（未探索）：完全不可见
- **已探索**（灰暗）：可见地形，不可见实体
- **可见**（明亮）：完全可见

**视野半径**：4.5 格（圆形欧几里得距离）

**灾变收缩**：随计时器进入后期阶段，地图边缘被黑暗吞噬（见 §五）。

## 五、计时器与灾变系统

### 5.1 四阶段递进

总时长 **480 秒（8 分钟）**：

| 阶段 | 名称 | 时间段 | 效果 |
|------|------|--------|------|
| `calm` | 灵气平和 | 0:00 - 4:00 | 安全探索，无压力 |
| `warning` | 灵气紊乱 | 4:00 - 5:30 | 计时器变金色，地图边缘开始收缩（0→30%） |
| `danger` | 瘴气弥漫 | 5:30 - 7:00 | 计时器变朱砂色闪烁，收缩加速（30%→70%） |
| `collapse` | 天崩地裂 | 7:00 - 8:00 | 地图快速坍塌（70%→100%），最后60秒 |
| `collapsed` | 时尽 | 8:00+ | **强制结算** |

### 5.2 灾变收缩公式

收缩 `progress`（0~1）驱动 `FogOfWar.collapseEdge(progress)`：

```
边框厚度 = floor(progress × min(地图宽, 地图高) × 0.4)
```

- `calm` 阶段：progress = 0（不收缩）
- `warning`：progress = 0 → 0.3
- `danger`：progress = 0.3 → 0.7
- `collapse`：progress = 0.7 → 1.0

### 5.3 强制结算惩罚

如果时间耗尽（`collapsed`）：
- 所有已捕获的灵契（异兽）**全部丢失**
- 灵石**保留 50%**
- 兽魂和天晶**归零**

### 6.4 接触方向系统（背刺机制）

异兽有**朝向角度**，玩家从不同方向接近会产生不同效果：

| 接触方向 | 角度差 | R/SR 反应 | SSR 反应 | 压制加成 |
|---------|--------|----------|---------|---------|
| **背后** | >120° | 无反应 | 无反应 | ✅ 背刺加成（ambush） |
| 侧面 | 60°-120° | 无反应 | 警觉 | 无 |
| 正面 | <60° | 警觉 | 立即逃跑 | 无 |

**背刺加成（ambush）效果**：
- 封灵成功率 **+20%**（上限 100%）
- 压制 QTE 目标区 **扩大 15%**（计时模式）
- 连点次数 **-2**，时限 **+0.5s**（连点模式）

## 七、线索调查系统

### 7.1 线索类型

地图生成时放置 **12 条线索**，最小间隔 3 格：

| 线索 | 名称 | 视觉 |
|------|------|------|
| `footprint` | 足迹 | 暖褐色兽爪印 |
| `resonance` | 共鸣 | 石青色脉冲环 |
| `nest` | 巢穴 | 碗状巢穴编织线 |

### 7.2 调查机制

- **触发距离**：玩家距线索 ≤1.2 格
- **基础调查时间**：2.0 秒
- **使用追迹灰**：调查时间降至 0.5 秒

### 7.3 线索阈值触发

| 累计线索数 | 触发事件 |
|-----------|---------|
| **3 条** | 生成一只 **SR** 品质异兽（距玩家 6-10 格随机方向） |
| **5 条** | 进行 **SSR 闪光判定**（见下方） |

### 7.4 SSR 闪光判定（Flash Roll）

```
总概率 = 基础概率(15%) + 线索加成(超过5条的每条+5%) + 封灵器加成(持有T4则+15%) + 保底加成
```

**保底加成**（来自 PitySystem）：

| 累计未获 SSR 次数 | 额外加成 |
|------------------|---------|
| 20-39 次 | +10% |
| 40-59 次 | +20% |
| 60-79 次 | +35% |
| **80+ 次** | **+100%（必出 SSR）** |

闪光判定失败则改为生成一只 SR。

## 八、压制 QTE 系统

### 8.1 触发条件

- 玩家距异兽 ≤1.5 格
- 点击"压制"交互按钮

### 8.2 计时模式（R / SR 品质）

**玩法**：一个指针在条形进度条上来回摆动，玩家需在指针进入目标区域时点击。

| 参数 | R 品质 | SR 品质 |
|------|--------|---------|
| 摆动速度 | 1.0 | 1.6 |
| 所需命中次数 | 1 | 2 |
| 目标区间 | [0.30, 0.70]（40% 宽度） | [0.35, 0.65]（30% 宽度） |
| 每次命中后速度 | ×1.15 | ×1.15 |
| 每次命中后区间缩减 | 两侧各 -0.02 | 两侧各 -0.02 |

**失败条件**：点击时指针不在目标区间 → 立即失败。

### 8.3 连点模式（SSR 品质）

**玩法**：限时内疯狂点击屏幕达到目标次数。

| 参数 | 默认值 |
|------|--------|
| 所需点击次数 | 8 次 |
| 时间限制 | 3.0 秒 |

**失败条件**：时间耗尽未达到目标次数。

### 8.4 道具 / 背刺加成

| 加成来源 | 计时模式 | 连点模式 |
|---------|---------|---------|
| 镇灵砂 | 目标区两侧各 +0.05 | 所需次数 8→6 |
| 背刺(ambush) | 目标区扩大 15% | 所需次数 -2（最低4），时限 +0.5s |

## 九、封灵（捕获）系统

### 9.1 封灵流程

```
压制成功 → 自动选择最高品级封灵器 → 掷骰判定 → 成功/失败
```

### 9.2 成功率计算

```
最终成功率 = 封灵器基础成功率 + 背刺加成(+20%)
上限 = 100%
```

| 封灵器 | 基础率 | +背刺 |
|-------|--------|-------|
| 素灵符 T1 | 75% | 95% |
| 青玉壶 T2 | 85% | 100% |
| 金缕珠 T3 | 92% | 100% |
| 天命盘 T4 | 98% | 100% |

### 9.3 封灵成功

- 创建灵契（contract）：`{beastId, type, name, quality, stable=false}`
- 播放 4.5 秒捕获动画（CaptureOverlay）
- 更新保底计数器：
  - 捕获 SSR → 重置 SSR 和 SR 计数器
  - 捕获 SR → 重置 SR 计数器，SSR 计数器+1
  - 捕获 R → 两个计数器各+1

### 9.4 封灵失败

- 封灵器被消耗（不退还）
- 异兽进入逃跑状态
- 显示"封灵失败"提示

### 9.5 捕获动画（CaptureOverlay）

4.5 秒不可跳过的封印演出，分 4 阶段：

| 阶段 | 时间 | 视觉效果 |
|------|------|---------|
| 墨染扩散 | 0-1s | 屏幕变暗，中心墨圈扩展 |
| 金光乍现 | 0.8-2s | 光芒辐射（R:4条 SR:8条 SSR:12条） |
| 符文浮现 | 1.8-3.2s | 粒子环绕（R:6个 SR:12个 SSR:24个），显示兽名首字 |
| 品质揭晓 | 2.8-4.5s | 完整兽名 + 品质标签，SSR 额外显示"天命之契.不朽灵印" |

## 十、撤离系统

### 10.1 撤离流程

```
到达撤离点(≤1.5格) → 站立等待 3 秒 → 灵契稳定性检定 → 不稳定灵契QTE → 结算
```

### 10.2 撤离引导

- 地图上有 **4 个撤离法阵**
- 玩家必须在撤离点范围内保持不动 3 秒
- 移动会中断撤离进度

### 10.3 灵契稳定性

撤离成功后，每个灵契独立判定是否不稳定：

| 品质 | 基础不稳定概率 | 使用归魂符后 |
|------|--------------|-------------|
| R | 5% | 0% |
| SR | 30% | 10% |
| SSR | 50% | 20% |

### 10.4 灵契稳定 QTE（ContractQTEOverlay）

对每个不稳定灵契：
1. **警告阶段**：显示"灵契震荡 (X/Y)"，闪烁朱砂光环
2. **QTE 阶段**：计时条，指针摆动速度 1.2，目标区 [0.25, 0.75]（50% 宽度）
3. **命中** → 灵契保住
4. **未命中** → 灵契**破碎**，该异兽永久丢失

## 十一、结算系统（ResultScreen）

### 11.1 结算触发

两种方式进入结算：
- ✅ **正常撤离**：撤离成功后
- ❌ **超时强制结算**：计时器耗尽

### 11.2 分数计算

| 来源 | 分值 |
|------|------|
| R 品质灵契 | +100 |
| SR 品质灵契 | +300 |
| SSR 品质灵契 | +1000 |
| 灵石（每个） | +2 |
| 兽魂（每个） | +10 |
| 天晶（每个） | +50 |
| 剩余时间加分 | +50/分钟 |
| 丢失 R 灵契 | -50 |
| 丢失 SR 灵契 | -200 |
| 丢失 SSR 灵契 | -500 |
| **最低分** | **0**（不会为负） |

### 11.3 结算动画

- 卷轴展开动画（1 秒，easeOutQuad）
- 逐条揭示获得的灵契和破碎的灵契
- 显示探索时长和最终分数
- 点击关闭回到大厅

### 11.4 数据结转

- 存活的灵契 → 录入图鉴（`GameState.bestiary`），更新品质收集记录
- 收集的资源 → 加入持久存储（灵石/兽魂/天晶）
- 统计数据更新：总探索次数、总捕获数、总撤离数

## 十二、保底系统（Pity）

### 12.1 双计数器

| 计数器 | 触发阈值 | 效果 |
|--------|---------|------|
| `ssrCount` | 80 次 | 保证下次 SSR 闪光判定成功 |
| `srCount` | 15 次 | 保证 3 线索时一定生成 SR |

### 12.2 SSR 渐进加成

| 累计失败次数 | 闪光概率加成 |
|-------------|-------------|
| 0-19 | +0% |
| 20-39 | +10% |
| 40-59 | +20% |
| 60-79 | +35% |
| **80+** | **+100%（保底）** |

### 12.3 计数器重置规则

- 捕获 SSR → **ssrCount 和 srCount 都归零**
- 捕获 SR → **srCount 归零**，ssrCount +1
- 捕获 R → 两个计数器各 +1

### 12.4 持久化

保底数据保存在 `saves/pity.json`，通过 HMAC 签名防篡改。

## 十八、技术系统

### 18.1 操控

- **虚拟摇杆**：屏幕 10%-80% 高度区域任意位置触发，半径 50px，死区 15%
- **交互按钮**：屏幕右下角，上下文敏感（压制/调查/收集/撤离）
- **点击交互**：ScreenManager 合成 tap 事件（偏移<15px，时长<0.4s）

### 18.2 相机

- 视野高度 10 格（固定）
- 指数平滑跟随：`lerp = 1 - exp(-5.0 × dt)`
- 视锥裁剪：仅渲染可见范围内的地形和实体

### 18.3 碰撞

- AABB 碰撞检测，实体碰撞盒 0.7×0.7 格
- 四方向滑动：X/Y 轴分离尝试 + 角落滑移（0.3 格偏移）

### 18.4 性能自适应

- 滚动窗口 90 帧采样
- FPS < 40 → 降画质（减少粒子、关闭大气效果、减少笔触细节）
- FPS > 55 → 升画质
- 3 个画质等级：Low / Medium / High

### 18.5 存档

- HMAC 签名校验防篡改
- 使用引擎 `File` 类写入沙箱目录
- 持久数据：`saves/gamestate.json`、`saves/pity.json`
