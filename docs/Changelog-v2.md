## 一、问题分析

### 1.1 图鉴不区分品质（P0）

**现状**：`BookScreen` 每只灵兽只显示一张卡片，只展示 `bestQuality`（最高品质）和总捕获次数。玩家无法看到：
- 我有哪些品质的白泽？（R? SR? SSR?）
- 每个品质捕获了几只？
- 还差哪些品质没集齐？

**根因**：`GameState.recordBeast()` 数据结构只有 `bestQuality` 和 `count`，没有分品质统计。

**修复方案**：
1. 扩展 `GameState.data.bestiary` 数据结构，增加 `qualities` 字段：
   ```lua
   bestiary["001"] = {
       discovered = true,
       captured = true,
       count = 5,           -- 总捕获数
       bestQuality = "SR",
       qualities = {         -- 新增：分品质统计
           R = 3,
           SR = 2,
           SSR = 0,
       }
   }
   ```
2. 修改 `GameState.recordBeast()` 记录分品质数据
3. 重新设计 `BookScreen` 卡片布局：
   - 每只灵兽卡片内展示三个品质槽位（R / SR / SSR）
   - 已获得的品质高亮显示 + 捕获次数
   - 未获得的品质灰显
   - 三品质全集齐时卡片有特殊边框

### 1.2 每日任务"获得1只异色"表述模糊（P1）

**现状**：任务描述"获得1只异色"容易误解为"奖励一只异色灵兽"。实际上：
- 任务目标：在探索中捕获1只SR+品质灵兽
- 任务奖励：追迹灰×5（不是灵兽）

**修复方案**：
1. 修改任务描述为更清晰的表述："捕获1只异色灵兽"
2. 奖励显示改为具体内容："追迹灰×5"

### 1.3 图鉴卡片视觉单调（P2）

**现状**：已收录灵兽只显示文字（品质标签 + 名称 + 描述 + 次数），缺乏水墨风格表现力。

**修复方案**：
1. 扩大卡片高度，容纳品质槽位信息
2. 每个品质用对应颜色的墨点/印章标识
3. 增加灵兽属性标签（火/水/雷等，用对应颜色小标签）
4. 三品质全集齐时，卡片边框用描金效果

## 二、具体实施

### 2.1 GameState.lua — 数据结构扩展

**修改 `recordBeast()` 函数**：
- 新增 `qualities` 表按品质分别计数
- 向下兼容旧存档（无 `qualities` 字段时自动初始化）

### 2.2 BookScreen.lua — 完全重写卡片渲染

**新卡片布局**（每张卡片高度从 10% → 15%）：

```
┌─────────────────────────────────────┐
│  [火] 玄狐                    ×5    │
│  追踪异兽之踪迹，以灵符封印        │
│                                     │
│  ● R ×3    ◎ SR ×2    ○ SSR ×0     │
│  ───────   ────────   ──────────   │
│  已获得     已获得      未获得      │
└─────────────────────────────────────┘
```

- **第一行**：属性标签 + 名称 + 总捕获次数
- **第二行**：灵兽描述
- **第三行**：三个品质槽位，各自显示状态和计数
  - 已获得：品质色墨点 + 次数
  - 未获得：空心圆 + "×0" 灰显

**三种卡片状态**：
| 状态 | 视觉表现 |
|------|---------|
| 未发现 | 灰色底，"???" 居中 |
| 见闻 | 淡墨底，名称半透明，无品质信息 |
| 已收录 | 暖色底，完整信息 + 品质槽位 |
| 全品质收齐 | 描金边框 + 特殊光效 |

### 2.3 DailyScreen.lua — 任务描述优化

将"获得1只异色"改为"捕获1只异色灵兽"。
