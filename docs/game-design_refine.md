## 一、游戏概述与设计哲学

### 1.1 基本信息

| 项目 | 内容 |
|------|------|
| **类型** | 水墨风异兽捕获 Roguelike（单局制提取玩法） |
| **视角** | 2D 俯视角 |
| **画面** | NanoVG 矢量渲染，水墨白描 + 朱砂点缀 + 宣纸底色 |
| **平台** | 竖屏移动端 |
| **单局时长** | 6～10 分钟（动态，取决于玩家策略） |

### 1.2 一句话描述

> 你是一名封灵师，在灵气崩解前深入灵境追踪、读懂、压制并封印异兽——每一只异兽都有自己的习性和破绽，带不走的，就永远留在灵境了。

### 1.3 三条设计原则

**原则一：每只异兽都应该"有脾气"**

异兽不是数值容器。它的属性、行为、QTE形式、捕后效果应该在玩法层面形成差异，让玩家对"今天遇到了风鸣"和"今天遇到了石灵"有完全不同的记忆。

**原则二：每一局都有一个"最难的决定"**

封灵师面临的核心张力是：时间有限，深入高危区收益更高但撤离风险更大。游戏应在准备、探索、撤离三个节点各提供一次真实的策略决策，而不只是"跑路"。

**原则三：图鉴填满是开始，不是结束**

收集30个图鉴词条是短期目标。后续应有赛季内容、挑战模式、异兽相性玩法等形式维持玩家的长期动力，使"封灵师"身份持续生长。

## 二、核心循环重设计

### 2.1 三层循环结构

```
【局内循环 · 8分钟】
  大厅准备（选灵境、选道具、选封印流派）
    → 灵境探索（读迷雾、判断路线、收集线索）
      → 异兽交锋（识破习性、背刺压制、QTE对决）
        → 撤离博弈（选时机、选路线、稳定灵契）
          → 结算入账

【日常循环 · 每天15分钟】
  每日任务 → 签到 → 赛季通行证进度 → 图鉴解锁

【长期循环 · 数周数月】
  封灵师境界成长 → 灵境叙事解锁 → 赛季新异兽 → 百灵志撰写
```

## 四、地图与探索

### 4.1 地图生成

- **地图尺寸**：20×30 格
- **出生点**：底部中央 `(10, 2)`，周围 3×3 区域强制为小路
- **撤离点**：4 个，分布在地图四角附近
- **结构倾向生成**：每张地图有一个主要"深入走廊"（偏左/偏右/中路随机），影响玩家最高效的深入路线

### 4.2 地形类型

| 地形 | 可通行 | 特殊效果 | 视觉 |
|------|--------|---------|------|
| 草地 `grass` | ✅ | 无 | 稀疏兰草弧线，翡翠底色 |
| 小路 `path` | ✅ | 移速 +10% | 暖色带状路面 + 踏痕 |
| 岩石 `rock` | 部分阻挡 | 大石不可通行 | 卵石点 + 皴纹 |
| 竹林 `bamboo` | 部分阻挡 | 密竹不可通行；**隐蔽玩家体型**（减少异兽感知范围2格） | 截面圆 + 放射叶 |
| 水面 `water` | ❌ | 完全不可通行 | 蓝色波纹 + 涟漪 |
| 瘴气 `danger` | ✅ | **每秒消耗1灵石**；处于瘴气中异兽感知范围 -1格 | 暗红漩涡 + 朱砂散点 |
| 边界墙 `wall` | ❌ | 地图边框 | 浓墨填充 + 皴纹 |

> **新增地形机制**：竹林赋予隐蔽效果、瘴气有灵石代价——地形从纯通行/阻挡升级为策略工具。

### 4.3 区域分布

| 区域 | Y 范围 | 特征 |
|------|--------|------|
| 安全区 | 深度 ≤15% | 35% 小路、50% 草地、15% 竹林，仅灵石 |
| 搜索区 | 深度 15%-40% | 混合地形，开始出现岩石，兽魂出现 |
| 稀有区 | 深度 40%-70% | 10% 瘴气、密集障碍，镇灵砂/兽目珠出现 |
| 高危区 | 深度 >70% | 25% 瘴气，天晶/归魂符/封印回响仅此处 |

### 4.4 资源分布

地图生成 **28 个资源节点**，每个产出 1-3 单位：

| 资源 | 权重 | 出现条件 |
|------|------|---------|
| 灵石 | 55 | 全图 |
| 追迹灰 | 18 | 全图 |
| 兽魂 | 12 | 深度 >40% |
| 天晶 | 5 | 深度 >70% |
| 镇灵砂 | 7 | 深度 >40% |
| 归魂符 | 5 | 深度 >70% |
| 兽目珠 | 4 | 深度 >40% |
| 封印回响 | 2 | 深度 >70% |

### 4.5 迷雾系统

三种状态：
- **黑暗**（未探索）：完全不可见
- **已探索**（灰暗）：可见地形，不可见实体
- **可见**（明亮）：完全可见，视野半径 **4.5格**

**竹林视野修正**：在竹林地形中，玩家视野半径降至 **3格**，但异兽对玩家的感知也相应削减。

**撤离路径指示**：进入`warning`阶段后，地图UI显示到最近撤离点的虚线指引（可关闭），避免玩家在灾变中迷失。

## 五、计时器与灾变系统

### 5.1 四阶段递进（调整节奏）

总时长 **480秒（8分钟）**，缩短`calm`阶段，让压力更早到来：

| 阶段 | 名称 | 时间段 | 效果 |
|------|------|--------|------|
| `calm` | 灵气平和 | 0:00 - 3:00 | 安全探索（从4分钟缩至3分钟） |
| `warning` | 灵气紊乱 | 3:00 - 5:00 | 计时器变金色，地图边缘收缩（0→30%），**撤离指引出现** |
| `danger` | 瘴气弥漫 | 5:00 - 7:00 | 计时器变朱砂闪烁，收缩（30%→70%），**瘴气地形伤害加倍** |
| `collapse` | 天崩地裂 | 7:00 - 8:00 | 快速坍塌（70%→100%），**全图异兽进入flee状态** |
| `collapsed` | 时尽 | 8:00+ | 强制结算 |

> **设计意图**：把calm缩短1分钟，给玩家更强的时间紧迫感；collapse阶段全图异兽进入逃跑态，暗示"此时追兽不合算"，引导玩家果断撤离。

### 5.2 灾变收缩

收缩公式不变，但新增**"瘴气涌动"**效果：`danger`阶段起，已收缩区域随机向内喷发瘴气粒子，视觉上比单纯黑幕压迫感更强。

### 5.3 强制结算惩罚（分层）

| 超时程度 | 惩罚 |
|---------|------|
| 超时 <30秒 | SSR灵契丢失，SR/R保留；灵石保留60% |
| 超时 30-60秒 | SSR/SR丢失，R保留50%；灵石保留40% |
| 超时 >60秒 | **全部灵契丢失**；灵石保留20% |

> **设计意图**：原版只有"全部丢失"一种惩罚等级，过于极端。分层惩罚让"超时一点点"和"彻底超时"体验有所区别，减少玩家的挫败感，同时保留撤离压力。

## 六、异兽系统（全面重设计）

### 6.1 核心设计原则：异兽有脾气

每种异兽除数值参数外，拥有：
1. **独有的QTE形式**（或显著的参数变体）
2. **属性被动效果**（对玩法产生实际影响）
3. **一条专属捕获技巧**（在图鉴中解锁）

### 6.2 异兽图鉴（10种，全面扩展）

#### 001 玄狐（火）

| 参数 | 数值 |
|------|------|
| 体型 | 0.42 |
| 速度 | 2.0 |
| 逃跑率 | 30% |
| 感知范围 | 3格 |

**属性被动**：玄狐被封印后，本局内调查线索速度额外 -0.3秒（火焰灼烧线索，加速解读）。

**QTE特殊规则**：计时条背景会随机出现"火焰扰动"——目标区间短暂跳变0.5秒，须等扰动消失后再点击。

**捕获技巧（图鉴解锁）**：从竹林地形接近玄狐，火焰感知被竹林压制，警觉距离从3格降至1.5格。

#### 002 噬天（暗）

| 参数 | 数值 |
|------|------|
| 体型 | 0.55（最大） |
| 速度 | 1.5 |
| 逃跑率 | 20% |
| 感知范围 | 4格 |

**QTE特殊规则**：压制条为**双指针同步模式**——屏幕同时出现两条压制条，须在同一秒内分别点击两条都命中才算成功（专为平板/双手操作优化，手机难度系数×1.3）。

**捕获技巧**：噬天有"黑暗护盾"——在已探索但非可见区域（灰暗区）内，逃跑率降低至5%；在完全可见区域内逃跑率上升至35%。

#### 003 雷翼（雷）

| 参数 | 数值 |
|------|------|
| 体型 | 0.50 |
| 速度 | 2.5 |
| 逃跑率 | 40% |
| 感知范围 | 4格 |

**QTE特殊规则**：SR/SSR雷翼的压制条会定期"电击抖动"——指针位置瞬间随机偏移，考验玩家的预判能力。

**捕获技巧**：等待雷翼停下后的1.5秒"静止窗口"再发起压制，可完全规避电击抖动。

#### 004 白泽（光）

| 参数 | 数值 |
|------|------|
| 体型 | 0.48 |
| 速度 | 1.8 |
| 逃跑率 | 30% |
| 感知范围 | 5格（最远） |

**属性被动**：白泽感知到玩家后不立即逃跑，而是朝玩家走近1格并"凝视"——此时玩家若移动，触发逃跑；若保持静止3秒，白泽主动降低警戒（可不用背刺直接压制，且背刺加成仍生效）。

**QTE特殊规则**：白泽压制条中央有"光晕稳定区"——指针穿过光晕区时短暂减速，给玩家更长的点击窗口，体现白泽温顺的特性。

**捕获技巧**：白泽是唯一可以"正面接触不触发逃跑"的异兽，但玩家须在凝视期间完全静止。

#### 005 石灵（土）

| 参数 | 数值 |
|------|------|
| 体型 | 0.45 |
| 速度 | 1.2 |
| 逃跑率 | 10% |
| 感知范围 | 3格 |

**属性被动**：石灵的线索（足迹）在地图上显示为石纹，**额外标记周边2格内的所有资源节点**（石灵熟悉地脉）。

**QTE特殊规则**：石灵压制条为**长条强力模式**——目标区很宽（60%宽度），但需命中3次，每次命中后有0.5秒"石化硬直"，指针停止移动，之后恢复并加速。

**捕获技巧**：石灵逃跑率极低，不必纠结背刺，但封灵失败后会进入"石化防御"状态——须等待5秒才能再次发动压制。

#### 006 水蛟（水）

| 参数 | 数值 |
|------|------|
| 体型 | 0.50 |
| 速度 | 2.2 |
| 逃跑率 | 40% |

**捕获技巧**：将水蛟从水面附近引离至草地或小路，可完全消除其速度加成，QTE难度回归标准。

## 七、线索调查系统

### 7.1 线索类型（扩展）

地图生成时放置 **14 条线索**，最小间隔 3 格：

| 线索 | 名称 | 视觉 | 特殊信息 |
|------|------|------|---------|
| `footprint` | 足迹 | 暖褐色兽爪印 | 调查后显示异兽大致方向 |
| `resonance` | 共鸣 | 石青色脉冲环 | 调查后显示异兽品质（R/SR/SSR） |
| `nest` | 巢穴 | 碗状编织线 | 调查后显示异兽种类 |
| `scentMark`（新） | 气息印记 | 半透明气雾漩涡 | 调查后揭示附近2格内的资源节点 |

> **设计意图**：不同线索提供不同维度的信息，玩家通过路线规划"拼出"异兽全貌，强化探索的解谜感。

### 7.2 调查机制

- **触发距离**：玩家距线索 ≤1.2 格
- **基础调查时间**：2.0 秒
- **使用追迹灰**：降至 0.5 秒

**线索信息联动**：调查3条不同类型线索（足迹+共鸣+巢穴各1条），额外获得"习性推断"——提示该异兽当前位于地图的大致象限。

### 7.3 线索阈值触发

| 累计线索数 | 触发事件 |
|-----------|---------|
| 3 条 | 生成一只 **SR** 品质异兽（距玩家 6-10 格随机方向） |
| 5 条 | 进行 **SSR 闪光判定** |
| 8 条以上 | 每多 1 条线索，下次 SSR 闪光判定基础概率 +3%（本局内累积） |

### 7.4 SSR 闪光判定（Flash Roll）

```
总概率 = 基础概率(15%)
       + 线索加成（超过5条的每条 +5%）
       + 封灵器加成（持有T4: +10%，T5: +20%）
       + 流派加成（追迹流派最高层: +10%）
       + 保底加成（见§十七）
```

3. 命中 → 灵契保住
4. 未命中 → 灵契**破碎**，该异兽永久丢失

## 十七、保底系统（重平衡）

### 17.1 双计数器

| 计数器 | 触发阈值 | 效果 |
|--------|---------|------|
| `ssrCount` | 40次（原80次） | 保证下次SSR闪光判定必成 |
| `srCount` | 10次（原15次） | 保证3线索时必生成SR |

> **重平衡原因**：原版80抽保底在8分钟/局的节奏下，最坏情况需640分钟（80局）。收紧至40抽，仍保持稀缺感，但周期缩短至合理区间（约320分钟 = 约40局）。

### 17.2 SSR渐进加成

| 累计失败次数 | 闪光概率加成 |
|-------------|-------------|
| 0-14 | +0% |
| 15-24 | +10% |
| 25-34 | +20% |
| 35-39 | +35% |
| **40+** | **+100%（保底）** |

### 17.3 计数器重置规则

- 捕获SSR → ssrCount和srCount都归零
- 捕获SR → srCount归零，ssrCount +1
- 捕获R → 两个计数器各 +1

### 17.4 持久化

保底数据保存在 `saves/pity.json`，通过HMAC签名防篡改。
